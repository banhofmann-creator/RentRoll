Implement Phase 9 of the RentRoll project: Output Channels & Integrations. This builds on 8 completed phases (226 tests, last commit 92774e9). Read CLAUDE.md and planning/PLAN.md Phase 9 plus planning/spec-architecture.md section 4 for full context.

What to build (6 steps):

Step 1 — Output Channel Plugin System (backend/app/channels/)

The channels/__init__.py already exists (empty). Create:

- base.py: Abstract OutputChannel base class with push(files, metadata) -> PushResult and test_connection() -> bool. Dataclasses: ExportFile (filename, content: bytes, file_type: str, category: str), ExportMetadata (stichtag, fund, properties: list[str], reporting_period_id), PushResult (success, channel, files_pushed, destination, errors). Follow the interface in spec-architecture.md section 4.
- local_filesystem.py: LocalFilesystemChannel — saves files to exports/{fund}/{stichtag}/ folder tree. Creates dirs as needed.
- registry.py: Channel registry with get_channel(name), list_channels(), register_channel(name, cls). Pre-registers local_filesystem.

Step 2 — Investor Reporting Pack Generator (backend/app/core/investor_pack.py)

Function generate_investor_pack(db, period_id, fund=None) -> (zip_bytes, filename, list[ExportFile]):
- Looks up the ReportingPeriod and its CsvUpload
- Generates BVI XLSX via generate_bvi_xlsx(db, upload_id, stichtag, is_draft)
- Generates portfolio overview PPTX via generate_portfolio_overview(db, upload_id)
- Generates lease expiry profile PPTX via generate_lease_expiry_profile(db, upload_id)
- Per fund: generates fund summary PPTX via generate_fund_summary(db, upload_id, fund_name)
- Per property in the fund(s): generates factsheet PPTX via generate_property_factsheet(db, upload_id, property_id)
- Wraps all into ExportFile objects, bundles into a ZIP
- If fund is None, include all funds from the upload
- Returns (zip_bytes, zip_filename, export_files_list)

The slide generators are in backend/app/core/slides.py and return io.BytesIO. The BVI export is in backend/app/core/bvi_export.py and returns bytes.

Step 3 — Export API (backend/app/api/export.py)

Create a FastAPI router (prefix="/export", tags=["export"]):

- GET /api/export/channels — returns list of available channels (name + description)
- POST /api/export/investor-pack?period_id=&fund= — generates and streams ZIP download
- POST /api/export/investor-pack/preview?period_id=&fund= — returns file manifest (filename, file_count, file list) without downloading
- POST /api/export/push — body: {period_id, channel, fund?}. Generates investor pack, pushes via channel, returns PushResult.

Register the router in backend/app/main.py following the existing pattern (import + app.include_router(export_router, prefix="/api")).
